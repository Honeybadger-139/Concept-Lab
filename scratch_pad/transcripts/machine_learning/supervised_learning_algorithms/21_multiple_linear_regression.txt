Session 1 – What is Machine Learning?
Definition & Overview
Machine learning teaches computers to learn patterns from data rather than explicit rules.
Everyday examples: search result ranking, face tagging, movie recommendations, voice
assistants, spam filtering.
Simple vs Technical Explanation
Simple: letting computers learn from practice data.
Technical: training a model to map input data to outputs (x → y).
Importance
Enables tasks that are impossible to hard-code, such as recognizing cats or detecting spam.
Session 2 – Why ML Is Widely Used & Why It Matters
Manual vs Learned Rules
Some problems (sorting) have explicit rules; others (speech recognition) require learned
patterns.
Examples from Industry
Speech recognition, mapping, fraud detection, self-driving cars, and industrial inspection use
ML.
Artificial General Intelligence (AGI)
ML today is “narrow AI”; general human-level intelligence remains far off.
Economic Impact
ML is projected to generate trillions in value across healthcare, manufacturing, finance, and other
sectors.
Session 3 – Defining Machine Learning & Types Overview
Formal Definition
ML is “the field of study that gives computers the ability to learn without being explicitly
programmed.”
Learning involves experience (E), task (T), and performance (P).
Supervised vs Unsupervised
Supervised learning uses labeled data (x, y) pairs.
Unsupervised learning uses unlabeled data to discover patterns.
Session 4 – Supervised Learning (Regression)
Regression Explained
Predicts continuous values (house prices, temperatures).
Basic example: fitting a line to house size vs price.
Terminology
Input feature: x; Output/target: y; Training set: collection of (x, y).
Model parameters (w, b) define the line: f(x) = wx + b.
Importance of Generalization
The goal is not memorizing training data, but performing well on unseen data.
Session 5 – Classification
Binary & Multi-class Classification
Predicts categories (e.g., benign vs malignant).
Uses decision boundaries to separate classes.
Multiple Features
Real-world models use multiple inputs (age, tumor thickness) to improve predictions.
Session 6 – Unsupervised Learning & Clustering
Clustering
Groups similar data points; examples: news article clustering, genetic data segmentation,
customer segmentation.
When to Use It
Use when labels are unavailable and you need to explore structure in the data.
Session 7 – Formal Unsupervised Learning, Anomaly Detection & Dimensionality
Reduction
Anomaly Detection
Identifies unusual data points (fraud, defects).
Dimensionality Reduction
Compresses high-dimensional data while preserving key information.
Session 8 – Jupyter Notebooks & Labs
Interactive Environment
Jupyter notebooks allow mixing code with text, running experiments line by line.
Optional & Practice Labs
Optional labs let you run provided code; practice labs let you write code yourself.
Session 9 – Linear Regression Pipeline
Process
Collect training data → choose model → train model → evaluate → deploy.
Model Parameters
Notation for examples (x^(i), y^(i)) and number of examples (m).
Session 10 – Supervised Learning Pipeline & Model Representation
Model as a Function
The model f(x) produces an estimate ŷ; the true value is y.
Parameters as Variables
w controls slope; b controls intercept.
Session 11 – Cost Function & Linear Regression
Squared Error Cost
J(w, b) = (1/2m) ∑ (f(x^(i)) − y^(i))².
Parameters (w, b)
Finding the best w and b minimizes the cost.
Session 12 – Cost Function Intuition
Visual Example
A simplified cost plot shows how different slopes affect error.
Goal: choose w and b that minimize the cost.
Session 13 – Visualizing the Cost Function in 3D & Contours
3D Surface & Contour Plots
In 3D, J(w, b) looks like a bowl; a contour plot shows “height rings.”
The lowest point (center) is the global minimum.
Session 14 – Relationship Between (w, b), Model & Cost
Examples
Poor choices of (w, b) correspond to lines far from data points and high cost.
Near the minimum, the line fits well and cost is low.
Session 15 – Gradient Descent Concept
Analogy
Imagine being on a hill and taking steps downhill in the steepest direction.
Local vs Global Minimum
Non-convex functions can have multiple valleys; linear regressionʼs squared error cost is convex.
Session 16 – Gradient Descent Update Rule & Learning Rate
Update Equations
w := w − α (∂J/∂w); b := b − α (∂J/∂b).
Learning Rate (α)
Controls step size; too small → slow convergence; too large → overshooting or divergence.
Simultaneous Updates
Update w and b together using their old values.
Session 17 – Intuition for the Derivative
Derivative Sign
Positive derivative means slope is upward → decrease w; negative → increase w.
Convergence
Near the minimum, the derivative approaches zero and updates become tiny.
Session 18 – Choosing the Learning Rate
Too Small vs Too Large
Too small: very slow progress. Too large: oscillation and divergence.
Fixed Rate Works
Because gradients shrink near the minimum, even a fixed α leads to convergence.
Session 19 – Final Linear Regression Algorithm
Derivatives
∂J/∂w = (1/m) ∑ (f(x^(i)) − y^(i)) x^(i)
∂J/∂b = (1/m) ∑ (f(x^(i)) − y^(i))
Batch Gradient Descent
Uses all training examples at every update step; guaranteed global minimum for convex cost.
Session 20 – Gradient Descent Demonstration
Visualization
Shows parameters moving along the contour plot toward the global minimum.
Batch vs Stochastic
Batch uses the entire dataset; stochastic uses one example per update (not covered here).
